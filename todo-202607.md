# MiniInflux.Net10 2026-07 生产化收敛清单

更新时间：2026-07-12

## 结论与范围

当前版本具备单节点的写入、WAL、segment、查询、压缩、离线备份和基础认证主链路，但还不能按生产级服务直接对外部署。本清单只覆盖小型**单节点**生产运行所需的正确性、安全性、可恢复性和可观测性收敛；不以多节点、复制或完整 InfluxDB 1.x 兼容为本月前置条件。

本次审计以源码和测试为准。已有 214 个 `[Fact]` / `[Theory]` 标记；`dotnet test .\\MiniInflux.Tests\\MiniInflux.Tests.csproj -nologo --no-restore` 可完成构建，但当前运行器没有打印完成后的 passed 汇总，不能把它当作生产验收证据。

## P0：上线前必须修复

### P0-1 认证端点判定反转

状态：`[done 2026-07-12]` 已改为统一授权判断；认证关闭和成功均可继续执行，失败返回 401/429，并有回归测试。

现状：`Program.cs` 的 `EnsureAuthorized` 在认证关闭时构造 401 后返回 `true`，认证成功时同样构造 401 后返回 `true`。所有调用点都是 `if (!EnsureAuthorized(...)) return authResult;`，因此会继续返回该 401；认证失败反而返回 `false` 并提前结束。`/write`、`/query`、`/metrics`、管理接口等受影响。

处理：修复共享的 `EnsureAuthorized` 返回语义：认证关闭或认证成功时返回 `true` 且不产生错误结果；失败时返回 `false` 并返回 401/429。保留 `/admin/api` 中间件的同一语义，不做各端点补丁。

验收：增加真实 HTTP 集成测试，覆盖 auth 关闭、正确 Basic、错误 Basic、限流后的 429；验证 `/write` 和 `/query` 至少各一条成功路径。

### P0-2 凭据泄漏与代理信任边界

状态：`[done 2026-07-12]` access log 已脱敏敏感 query 参数；query 参数认证默认关闭；只有配置的可信代理才读取 `X-Forwarded-For`。

现状：访问日志拼接完整 query string，而认证支持 `?u=...&p=...`，默认访问日志会把密码写入日志；认证限流直接信任任意 `X-Forwarded-For`，外网客户端可伪造地址绕过限流或污染审计。

处理：日志对 `u`、`p` 及未来敏感参数统一脱敏；默认关闭 query 参数认证，或至少明确标记为仅兼容模式；仅当显式配置可信反向代理时才解析转发头，否则使用连接 IP。

验收：请求带 `p` 后应用日志和 access log 均不含密码；伪造 `X-Forwarded-For` 不改变未配置可信代理时的限流身份。

### P0-3 安全默认值与传输加密

状态：`[done 2026-07-12]` 仓库配置不再含可用密码；认证或 TLS 关闭时启动会记录 Warning，认证启用时 Production 仍拒绝占位符密码，TLS 启用时只监听 HTTPS。

现状：仓库配置含示例超级管理员口令；启用 TLS 时仍保留 HTTP listener；TLS、认证和对外监听之间没有生产启动校验。

处理：移除仓库中的可用口令，支持从环境变量/secret 文件注入；认证/TLS 关闭时明确记录风险 Warning；认证启用时生产模式拒绝占位符口令；TLS 启用时显式仅 HTTPS 监听，并记录证书加载失败。

验收：认证/TLS 关闭时服务可启动且产生 Warning；启用认证时缺少 secret、生产环境使用占位符密码会启动失败；有效 TLS 配置下写入和查询只能通过 HTTPS 成功。

### P0-4 写入持久性和磁盘错误不得静默失败

状态：`[done 2026-07-12]` WAL、flush、retention 和 compaction 的后台异常已进入存储健康状态和 Prometheus 指标；WAL 持久化失败时写入返回失败，健康检查为 503，成功 fsync 后恢复可写。

现状：WAL 定时 `FlushToDisk`、保留策略清理、compaction 等后台路径吞掉异常；磁盘满、权限变化或 fsync 失败不会暴露给 `/health`、指标或告警，可能在表面健康时丢失数据/积压任务。

处理：后台任务捕获异常后记录结构化错误、失败计数、最后失败时间和健康状态；连续 WAL 持久化失败时停止接收写入并返回 503，恢复成功后再解除。不要吞掉数据路径异常。

验收：注入 fsync/flush/compaction 失败后，health 变为不健康、指标可见，写入按策略被拒绝；恢复后有明确恢复事件。

## P1：上线前必须验证或实现

### P1-1 有界资源与过载保护

状态：`[done 2026-07-12]` HTTP 写入已接入现有有界 `WriteQueue`，队列压力返回 429；Production 启动要求查询时限和内存上限为非零。

`WriteQueue` 已实现但 HTTP 目前直接调用 `WriteInternalAsync`，`Write.QueueCapacity` 与 `BatchSize` 没有进入在线写入路径。确认后选择其一：接入现有队列并返回明确 429/503，或删除无效配置和未使用类。查询的 `MaxQueryDurationMs`、`MaxQueryMemoryBytes` 默认均为 0，也要给生产 profile 提供非零安全值。

验收：并发写入超过容量时服务延迟和内存有上限；慢查询、超行数、超内存估算分别得到受控错误并产生指标。

### P1-2 备份、恢复和灾难演练

状态：`[done 2026-07-12]` 在线管理接口只接受 `Data.BackupDir` 下的相对目录名；未配置该目录时接口拒绝执行，CLI 仍保留离线运维用途。

当前备份是 `FlushAll` 后目录复制并校验 hash，不是在线一致性快照；管理 HTTP 允许管理员提交任意本机路径。先限制该接口到本机/禁用，或改为受控备份目录；生产上以 CLI 离线执行。补一套可重复演练：备份、校验、空目录恢复、启动恢复、抽样查询校验和回滚。

验收：每个发布版本在干净目录完成恢复演练；破损备份、空间不足、恢复中断和 pending restore 都能安全失败且保留原数据。

### P1-3 健康检查、指标与告警基线

状态：`[done 2026-07-12]` health 已纳入 WAL 状态和最小剩余磁盘阈值；Prometheus 已导出写路径可用性、存储失败数和数据卷剩余空间；Production 要求设置磁盘阈值。

`/health` 当前恒为 ready，无法反映 WAL、磁盘、恢复、flush、compaction backlog、CQ 连续失败或剩余空间。扩展 readiness/liveness，并导出上述指标和版本/配置摘要（不得泄露 secret）。

验收：可用一组 Prometheus 规则告警：写入失败、WAL/fsync 失败、队列/缓冲逼近上限、compaction backlog、磁盘空间、CQ 连续失败、查询超时。

### P1-4 故障与长时间一致性测试

状态：`[done 2026-07-12]` 已补可执行的 `scripts/soak-test.ps1`，默认运行 24 小时写入/查询混合 workload 并输出可归档结果。发布前仍必须在目标环境实际运行，不能以脚本存在替代演练证据。

补黑盒 HTTP 测试和进程级故障注入：写入中断电、WAL 截断/CRC 错误、flush/manifest/compaction 中断、恢复后重复点 last-write-wins、删除和 compaction 并发。至少完成 24 小时 soak（持续写入、乱序、删除、查询和压缩混合），保存压测原始输出但不提交 `.benchmarks/`。

验收：测试输出包含写入数、查询错误、恢复结果、数据校验和、内存/磁盘/延迟趋势；无未解释的数据差异、死锁或无界增长。

## P2：性能与运维优化（先量化再做）

### P2-1 写入路径

状态：`[done 2026-07-12]` 写入队列已导出 pending 指标；本轮仅在接入队列后做基线压测，不在未见并发瓶颈时增加锁分片或新缓存。

最新 Release 基线（`100,000 points / batch 5,000 / concurrency 1 / query median 3 次`）：write `82,538.10 pts/s`，flush `2,251.39 ms`，aggregate `1.79 ms`，raw LIMIT 1000 `15.59 ms`。队列首轮使单并发写入降到约 `51k pts/s`，已用空闲直写快路恢复；高并发场景仍走有界队列。

现有 100k 点基线约 104k pts/s，优先完成 P1-1 后再测。按 profile 决定是否保留队列批处理、调低 `FlushAll` 停顿、按数据库/RP 分片锁降低全局锁竞争；不要在没有并发 profile 前新增缓存、线程池或复杂 WAL 格式。

验收：并发 1/4/16、多 measurement、高基数和乱序 workload 分别记录 p50/p95/p99、吞吐、GC、flush 与恢复耗时，并与当前基线比较。

### P2-2 压缩与磁盘生命周期

状态：`[done 2026-07-12]` compaction stats 已补 backlog 任务估算；compaction 仍全局串行执行，作为明确的单节点 I/O 并发上限。

补 compaction backlog、每层 segment 数/字节数、耗时、失败与 tombstone 积压指标，增加 I/O 限速/并发上限。再根据真实 segment 和查询 profile 决定是否需要列块索引、更多 codec 或预聚合；当前 raw LIMIT 已经下推，不为假设收益重写存储引擎。

### P2-3 日志与运行配置

状态：`[done 2026-07-12]` access log 已带 `X-Request-Id`，文件日志支持大小滚动与保留数量配置；`WriteTracing` 继续默认关闭。

增加日志滚动、保留策略、结构化 JSON 输出和 request/correlation id；把开发、生产配置分开，给每个资源阈值写出默认值和含义。`WriteTracing` 必须继续默认关闭，并明确其可能记录业务数据。

## P3：按业务需求再投入

- 外部 secret provider、无停机凭据/证书轮换和细粒度 RBAC。
- 在线一致性快照（快照屏障、WAL 截点、manifest 一致性协议）。
- 系统化 InfluxDB 1.x HTTP/InfluxQL corpus 兼容矩阵。
- 多副本、集群、跨节点恢复和多租户隔离。

## 生产准入门槛

在 P0 全部完成、P1-1 至 P1-4 有可复现证据前，不将本项目标记为生产级。首次生产部署限定为：单节点、私有网络或受控反向代理、HTTPS、外部 secret、固定资源阈值、定期恢复演练和监控告警已接入。
