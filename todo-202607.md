# MiniInflux.Net10 2026-07 性能优化清单

更新时间：2026-07-04

## 最新压测基线

口径：`100,000 points / batch size 5,000 / concurrency 1 / query median 3 次`，脚本：

```powershell
.\scripts\compare-with-influxdb.ps1 -Points 100000 -BatchSize 5000 -Concurrency 1 -QueryIterations 3
```

结果：

```text
MiniInflux：write 104,401.50 pts/s，flush after write 2,025.28 ms，aggregate 1.69 ms，raw LIMIT 1000 14.44 ms
InfluxDB 1.7.9：write 118,199.34 pts/s，aggregate 9.02 ms，raw LIMIT 1000 10.11 ms
MiniInflux storage：SegmentFiles=1，SegmentBytes=14,141，WalBytes=6,760,520
MiniInflux aggregate report：UsedAggregatePushdown=true，UsedSeriesIndexPushdown=true，SegmentMetadataFooterHits=1，DurationMs=0
MiniInflux raw report：ScannedPoints=1,000，RowsReturned=1,000，DurationMs=10，UsedSeriesIndexPushdown=true，UsedStreamingRawSelect=true，SegmentColumnsRead=2，PointsMaterialized=0，LimitPushdownStopReason=segment-limit
```

结论：

```text
简单聚合继续优于本轮 InfluxDB 对照。
raw LIMIT 1000 已命中 LIMIT pushdown 和多字段列直写：内部 report 显示只扫描 1,000 points，Point 物化为 0。
flush after write 已到 2.0s 左右；SegmentWriter 已避开 flush 时重新构造 series key、用一次遍历建列，并去掉每列 timestamp/value 数组复制，文件大小仍保持 14KB 级别。
```

## 下一轮优化计划

目标：优先缩小 raw LIMIT 1000 差距，随后降低 flush 成本。暂不重写存储引擎，不引入重量级依赖。

```text
P0-1. [done] raw DESC LIMIT 跨 segment 单 series 快路：普通查询和 fast JSON 共用按 segment metadata MaxTime 倒序读取，达到 LIMIT 后停止。
P0-2. [done] raw LIMIT 输出路径减少物化：已为 flushed 单 series、多字段等长列增加列直写 JSON/QueryResponse，Point 物化降为 0；不满足条件时回退旧路径。
P0-3. [done] raw query report 增加 SegmentColumnsRead / PointsMaterialized / LimitPushdownStopReason，方便确认是否真在 LIMIT 后停读。
P1-1. [done] flush path 减少 shard/column 分组成本：FlushLocked 改为一次遍历归桶，SegmentWriter 复用已缓冲 SeriesKey 并一次遍历建列，避免 LINQ GroupBy/SelectMany/重复 SeriesKey.From，同时去掉每列 timestamp/value ToArray。
P1-2. [done] WAL checkpoint / tombstone / manifest 的 SaveIfDirty 边界复查：显式 FlushAll/FlushDatabase/PeriodicFlush 已合并 WAL checkpoint；manifest/tombstone 保持必要写入。
P2-1. [deferred] measurement -> series -> shard/segment 的持久化细粒度映射：当前 raw report 只读 2 个 segment columns，目录和 metadata 筛选不是瓶颈，暂不推进。
P2-2. [deferred] 列块级 min/max 与后台预聚合：当前单 segment 基线 aggregate 已 4 ms 内，raw 已 LIMIT 停读；不为当前口径提前加复杂度。
```
